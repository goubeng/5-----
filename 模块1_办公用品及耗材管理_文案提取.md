## 页面 2-1｜业务现状

| 文案类型 | 文案内容 |
|---|---|
| 页面标题 | 2.1 业务现状 |
| 页面副标题 | 解决办公用品及耗材管理中“需求不准、额度滞后、配送无痕、统计耗时”的问题。 |
| 痛点 1 标题 | 需求收集靠人工 |
| 痛点 1 内容 | 需求靠表格收集，进度需人工催报汇总。 |
| 痛点 2 标题 | 额度管控不前置 |
| 痛点 2 内容 | 申领时难掌握额度占用，常到报销才发现额度不足。 |
| 痛点 3 标题 | 配送签收靠线下 |
| 痛点 3 内容 | 配送签收、清单核对依赖线下确认，过程留痕不足。 |
| 痛点 4 标题 | 报销统计靠核对 |
| 痛点 4 内容 | 报销统计需人工反复比对单据，耗时且口径易不一。 |
| 页面总结 | 归根结底，问题不是没有流程，而是流程分散、数据分散、过程难追、统计靠人工。 |

## 页面 2-2｜建设方案

| 文案类型 | 文案内容 |
|---|---|
| 页面标题 | 2.2 建设方案 |
| 板块 1 标题 | 申领前形成“依据”闭环 |
| 板块 1 流程 | 近三年消耗分析 → 年度需求填报 → 采购目录发布 → 年度额度生成 |
| 板块 1 说明 | 基于各单位近三年历史消耗和年度均值辅助需求填报，形成后续目录、单价和年度额度依据。 |
| 板块 2 标题 | 申领后形成“执行”闭环 |
| 板块 2 流程 | 线上申领 → 额度校验 → 审批流转 → 供应商配送 → 清单统计 |
| 页面总结 | 前置测算解决“能不能申领”，执行闭环解决“怎么申领、谁审批、谁配送、怎么核对”。 |
| 系统联动说明 | 系统联动：与财务系统关联，提前冻结申报额度；锁总额，不锁每单。供应商侧仅展示配送必要信息，结果回传内网留痕。 |

## 页面 2-3｜预期成效

| 文案类型 | 文案内容 |
|---|---|
| 页面标题 | 2.3 预期成效-人工到管控 |
| 页面副标题 | 通过任务分发、额度校验、清单生成和统计沉淀，减少人工催报、反复核对和事后补账。 |

### 预期成效对比表

| 指标 | 原有方式 | 系统建设后 | 预期变化 |
|---|---|---|---|
| 需求催收 | 30 余个单位逐一催报 | 系统任务分发，填报进度可视 | 减少人工催报和汇总 |
| 报销频次 | 约 4 次 / 年 | 1—2 次 / 年 | 报销组织频次下降 |
| 核对周期 | 单次约半个月 | 1—2 天 / 次 | 核对周期明显缩短 |
| 发票清单 | 约 500 余张 / 季度 | 系统生成清单和消耗统计 | 减少整理、比对和纸张消耗 |

| 文案类型 | 文案内容 |
|---|---|
| 页面总结 | 预期效果：减少催报、核对、纸质整理和纸张消耗，提升额度管控与统计支撑能力。 |

## 页面 2-4｜场景演示

| 文案类型 | 文案内容 |
|---|---|
| 页面标题 | 2.4 场景演示 |
| 页面副标题 | 基于年度需求测算、采购目录和年度额度，形成从线上申领、审批配送到清单核对和消耗统计的执行闭环。 |
| 场景说明 | 解决申领中“怎么领、能不能领、谁来批、谁配送、怎么统计”的问题。 |
| 页面流程 | 线上申领 → 审批流程 → 供应商配送 → 送达确认 → 清单打印 → 消耗统计 |

### 场景演示流程步骤表

| 流程步骤 | 步骤名称 | 文案内容 |
|---|---|---|
| 步骤 1 | 采购目录浏览 | 申领单位基于已发布且生效的采购目录查看可申领用品，按品目类别、用品名称、协议单价和供应商筛选所需办公用品及耗材。 |
| 步骤 2 | 我的申领车 | 经办人将所需用品加入申领车，系统自动计算申领数量、协议单价和申领金额，提交前可统一核对申领明细。 |
| 步骤 3 | 申领提交与额度校验 | 申领提交时，系统自动校验年度剩余额度和季度浮动阈值；年度剩余额度不足时阻断提交，超过季度浮动阈值时标记为超额度申领并进入升级审批。 |
| 步骤 4 | 单位审批 | 申领单提交后进入单位审批，单位负责人审核本单位申领内容。未触发超额度规则的申领，审批通过后进入供应商配送环节。 |
| 步骤 5 | 超额度审批 | 超过季度浮动阈值的申领，经单位审批通过后升级至警保部审批，由警保部结合额度执行情况和实际保障需要进行审核。 |
| 步骤 6 | 供应商配送单生成 | 申领审批通过后，系统按供应商自动拆分配送单，内网侧保留申领单位、用品明细和配送状态，供应商侧生成待接单任务。 |
| 步骤 7 | 供应商接单与配送 | 供应商在互联网侧移动端接收配送任务，仅查看配送品目、数量和必要配送信息，接单后反馈预计送达时间。 |
| 步骤 8 | 送达确认与状态回传 | 供应商完成配送后确认送达，配送状态回传至内网业务系统，申领单位和管理端可查看配送进度、送达结果和订单履约情况。 |
| 步骤 9 | 管理端追踪 | 管理端可对供应商配送进度、送达结果和订单履约情况进行追踪。 |
| 步骤 10 | 使用清单与配送单打印 | 系统根据申领和配送结果生成使用清单和配送单，支持预览打印，便于签收确认、归档核对、报销支撑和后续统计。 |
| 步骤 11 | 消耗统计与履历留痕 | 系统对申领数量、消耗金额、用品类别、单位使用情况、额度执行情况和配送履约情况进行统计分析，申领、审批、配送、送达等关键节点形成消息通知和操作履历。 |
| 步骤 12 | 申领与额度执行分析 | 警保部可跟踪各单位申领、消耗和额度执行情况。 |
| 步骤 13 | 历史资金使用分析 | 支持各单位近三年资金使用情况分析，为下一年度需求测算提供参考。 |
| 步骤 14 | 供应商履约跟踪 | 支持配送订单状态和供应商履约情况跟踪。 |
